КНИГА ПРОЕКТА «СЛОВО СИБИРИ»
Версия: 1.0
Формат: техническо-проектное описание простым языком

==================================================
1. О ЧЕМ ЭТОТ ПРОЕКТ
==================================================

«Слово Сибири» — это учебная веб-платформа (MVP) для изучения бурятского языка.
Проект строится вокруг идеи: короткие понятные уроки + сразу практика + фиксация прогресса.

Главная цель проекта:
- дать ученику удобный путь обучения;
- дать преподавателю инструменты создания курсов и уроков;
- дать администратору контроль пользователей и модерацию.


==================================================
2. КАК УСТРОЕН ПРОЕКТ В ЦЕЛОМ
==================================================

Проект построен на Laravel (PHP) и включает:
- backend (маршруты, контроллеры, сервисы, бизнес-логика);
- frontend (Blade-шаблоны, CSS, JavaScript);
- ролевую модель доступа (student / teacher / admin);
- систему учебного контента (курс -> модуль -> урок);
- интерактивные механики обучения и обсуждений.

Ключевая архитектурная особенность:
часть экранов работает через единый маршрут teacher/indes с параметром page,
где уже выбирается нужный экран интерфейса.


==================================================
3. РОЛИ ПОЛЬЗОВАТЕЛЕЙ И ИХ ВОЗМОЖНОСТИ
==================================================

3.1 Студент
- Входит в систему и попадает в личный кабинет.
- Открывает каталог курсов.
- Выбирает курсы, проходит уроки.
- Выполняет задания и получает прогресс/XP.
- Может оставлять отзывы и участвовать в обсуждениях.

3.2 Преподаватель
- Создает и редактирует курсы, модули, уроки.
- Настраивает структуру и содержание.
- Просматривает рабочие разделы преподавателя.

3.3 Администратор
- Управляет ролями пользователей.
- Работает с модерацией (включая жалобы/контент).
- Контролирует административный раздел.


==================================================
4. ОСНОВНЫЕ РАЗДЕЛЫ И МАРШРУТЫ
==================================================

Публичные страницы:
- /                      главная

Админ:
- /admin


==================================================
5. ПРИНЦИП РАБОТЫ БЭКЕНДА
==================================================

5.1 Входные точки
- routes/web.php описывает HTTP-маршруты.
- Middleware ограничивают доступ по ролям.

5.2 Контроллеры
В проекте используются контроллеры для:
- авторизации;
- администрирования;
- учебного контура преподавателя/студента;
- форума и комментариев урока;
- API для квеста/новеллы.

5.3 Сервисы
Сервисный слой отвечает за:
- подготовку данных для дашбордов;
- логику достижений студента;
- агрегацию и преобразование данных перед выдачей в шаблоны.

5.4 Бизнес-логика
Логика обучения построена вокруг последовательности:
курс -> модуль -> урок -> шаги -> результат.

При выполнении действий пользователя:
- фиксируется состояние;
- обновляется прогресс;
- может начисляться XP и достижения;
- данные отображаются в личном кабинете.

==================================================
7. МИНИ-НОВЕЛЛА (КВЕСТ)
==================================================

Мини-новелла — отдельный интерактивный режим обучения.

Как работает:
1. Страница /quest запрашивает сценарий через API.
2. JS-движок загружает текущую сцену.
3. Пользователь выбирает варианты ответа.
4. Движок переключает узлы истории (с учетом переходов).
5. Параллельно работает словарь сцены и подсветка слов.
6. Показывается прогресс прохождения и конечные состояния.

Зачем это нужно:
- повышает вовлеченность;
- показывает, как учебный контент можно подавать через сюжет;
- делает обучение менее «сухим».


==================================================
8. ДАННЫЕ И СОСТОЯНИЕ
==================================================

На уровне сущностей проект опирается на:
- пользователей;
- курсы;
- модули;
- уроки;
- прогресс;
- достижения;
- отзывы;
- обсуждения/комментарии.

Типовой цикл данных:
1. Пользователь выполняет действие.
2. Контроллер принимает запрос.
3. Сервис/модель обновляет состояние.
4. Система возвращает обновленный экран/статус.
5. Пользователь видит результат (например, новый прогресс).


==================================================
9. БЕЗОПАСНОСТЬ И ДОСТУПЫ
==================================================

В проекте применяются базовые механизмы:
- CSRF-защита форм;
- авторизация;
- middleware по ролям;
- ограничение части запросов (throttle) для чувствительных действий.

Это снижает риск несанкционированного доступа и перегрузки некоторых API.


==================================================
10. СИЛЬНЫЕ СТОРОНЫ MVP
==================================================

- Четкое разделение ролей.
- Рабочий цикл «контент -> обучение -> прогресс».
- Интерактивные элементы и обсуждения.
- Масштабируемая структура курса (иерархия).
- Демонстрационный сценарий (мини-новелла).


==================================================
11. ОГРАНИЧЕНИЯ ТЕКУЩЕЙ ВЕРСИИ
==================================================

Как и любой MVP, проект имеет ограничения:
- часть экранов объединена в один маршрутный хаб;
- визуальная консистентность еще требует шлифовки;
- аналитика может быть глубже (по времени, ошибкам, траекториям);
- модерационный контур можно расширить.


==================================================
12. КУДА РАЗВИВАТЬ ПРОЕКТ ДАЛЬШЕ
==================================================

Рекомендуемые шаги:
1. Вынести ключевые разделы в отдельные чистые URL.
2. Усилить учебную аналитику (тепловые карты ошибок, динамика по темам).
3. Добавить персональные траектории и рекомендации заданий.
4. Доработать модерацию и журнал действий.
5. Подготовить мультиязычность интерфейса.
6. Добавить автотесты критических сценариев (E2E + feature).


==================================================
13. КОРОТКОЕ РЕЗЮМЕ
==================================================

«Слово Сибири» — это работающий MVP образовательной платформы,
в котором уже реализованы:
- ролевой доступ,
- управление учебным контентом,
- прохождение курсов,
- фиксация прогресса,
- интерактивный демонстрационный режим.

Проект пригоден как для защиты, так и для дальнейшего роста до
полноценного учебного продукта.
